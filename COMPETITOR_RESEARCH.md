## Market Landscape Summary

| Tool | Starting Price | Target | Key Weakness |
|------|---------------|--------|--------------|
| **Canny** | $24/mo (25 users) → $99/mo (Pro) | Startups → Enterprise | Complex tracked-user pricing, expensive |
| **Nolt** | $29/mo | Small-mid businesses | Limited features, "outgrow it" concern |
| **Featurebase** | Free → $49/mo | All sizes | Feature-rich = complex, aggressive marketing |
| **Frill** | $25/mo (50 ideas limit) | Small teams | Idea limits feel restrictive |

| **Sleekplan** | $13/mo | Budget-conscious | Less polished |
| **Rapidr** | $49/mo | Startups | Mid-market positioning |

## Tally's Winning Formula (Applied to Feedback Tools)

### What Made Tally Win:
1. **Free tier that's genuinely useful** - No credit card, build forms without signup
2. **One pricing tier** - No upsells, no dark patterns, radical simplicity
3. **Design-first** - Notion-inspired aesthetic, text-editor interface
4. **Opinionated positioning** - Built for "people like us" (indie hackers, no-code makers)
5. **"Made with Tally" viral loop** - Free users spread the word
6. **Community-centric** - Daily user engagement, built in public

### Current Market Gaps (Tally Method Applied):

**Gap 1: No "Tally for Feedback" exists**
- Canny's free tier limits to 25 tracked users (useless for validation)
- Frill limits active ideas to 50
- No tool offers "unlimited free tier with branding badge"

**Gap 2: Pricing complexity everywhere**
- Canny just switched to confusing "tracked user" model
- Most tools have 3-4 tiers with feature gates
- Nobody offers simple "one plan, everything included"

**Gap 3: Enterprise creep**
- Every tool is "startups to enterprise" positioning
- None specifically target solo indie makers
- Features bloat to serve bigger customers

**Gap 4: Design mediocrity**
- Functional but not beautiful
- Nobody has Linear/Notion-level polish
- Opportunity for design as differentiator

## Recommended Positioning: "The Anti-Canny"

### Your Angle (From Playbook):
> "Why would they switch to you?"

**Target:** Solo founders and tiny teams (1-5 people) who:
- Are building in public
- Want beautiful tools (design-conscious)
- Hate complex pricing
- Don't need enterprise features

**Positioning Statement:**
"Simple feedback for indie makers. No tracked users. No feature gates. Just feedback, voting, roadmap, and changelog—beautifully designed."

### Differentiation via Subtraction:

| Competitors Add | You Remove |
|-----------------|------------|
| AI autopilot features | Just manual, intentional |
| Jira/Salesforce integrations | Simple export/webhook |
| User segmentation | Everyone sees everything |
| Complex permissions | One admin, public board |
| Multiple boards | One board per project |

## Chosen Strategy: Anti-Canny + Tally Viral Model

### Combined Positioning
**Tagline idea:** "The simplest way to collect feedback. Free forever with badge, $9/mo without."

**Core Principles:**
1. Radically simple (fewer features than competitors)
2. Beautiful by default (design as differentiator)
3. Generous free tier (viral "Powered by" growth engine)
4. One paid plan (no pricing page anxiety)

### Build-First Approach: MVP Feature Set

**Must Have (Core Loop):**
- [ ] Feedback submission form
- [ ] Upvoting on feedback items
- [ ] Public board view (SEO-indexable)
- [ ] "Powered by FeedbackApp" badge on free tier
- [ ] Status labels (Planned, In Progress, Done)

**Nice to Have (Post-MVP):**
- [ ] Public roadmap (Kanban view)
- [ ] Changelog / announcements
- [ ] Email notifications
- [ ] Custom domain (Pro only)
- [ ] Remove badge (Pro only)

**Explicitly NOT Building:**
- No AI features
- No Jira/Salesforce integrations
- No complex permissions/roles
- No user segmentation
- No multiple boards per project

### Pricing (Ship With)
```
Free: Full features + "Powered by FeedbackApp" badge
Pro: $9/mo - Remove badge, custom domain
```

### Success Signals (Post-Launch)
- Users return unprompted
- Organic sign-ups via "Powered by" badges
- Users ask for specific features (not just "more stuff")
- Someone upgrades to Pro to remove the badge

### Kill Criteria (Per Playbook)
- Crickets after 2-3 weeks live
- No organic sharing
- Zero Pro conversions after 50+ free boards
